# Homepage Setup

## Overview

Homepage is used as a central dashboard for my home lab. It provides a simple way to organize and access self-hosted tools, monitoring platforms, and local services from one place.

## Purpose

The goal of this setup is to create a clean landing page for my home lab environment and improve visibility into the tools and services I am running locally.

## Current Use Cases

* Central dashboard for home lab applications
* Quick access to self-hosted services
* Organization of monitoring and infrastructure tools
* Foundation for future documentation and automation workflows

## Services Listed

Current or planned services may include:

* Uptime Kuma
* Proxmox
* NAS services
* Docker-hosted applications
* Local AI tools
* Future cybersecurity lab tools

## Security Considerations

* Sensitive tokens, usernames, passwords, and API keys are not included in public documentation
* Internal URLs and private IP addresses are removed from public examples
* Screenshots are reviewed before being uploaded
* Public exposure is avoided unless a service is intentionally secured
* Configuration files should be reviewed before being committed to GitHub

## What I Learned

* How to organize self-hosted applications in a dashboard
* How YAML configuration files are used to customize services
* How documentation helps make a lab easier to maintain
* Why sensitive information should be removed before publishing files publicly
