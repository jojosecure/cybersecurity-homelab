# Home Lab Roadmap

This roadmap outlines the planned growth of my cybersecurity home lab. The goal is to build hands-on experience with infrastructure, monitoring, security tooling, automation, and local AI capabilities.

## Phase 1: Foundation

* Set up Raspberry Pi for lightweight services
* Deploy Docker and Docker Compose
* Set up Uptime Kuma for monitoring
* Set up Homepage as a central dashboard
* Document all major setup steps in GitHub

## Phase 2: Security Improvements

* Review exposed services and reduce unnecessary access
* Improve secrets management for configuration files
* Add backup and restore documentation
* Document secure configuration practices
* Add basic logging and monitoring notes
* Review screenshots and files before uploading to GitHub

## Phase 3: Infrastructure Expansion

* Add NAS-based services
* Explore Proxmox for virtualization
* Create network diagrams
* Document hardware and software inventory
* Organize Docker Compose files by service

## Phase 4: Cybersecurity Practice Lab

* Add vulnerability scanning tools
* Add log analysis or SIEM tooling
* Create intentionally vulnerable lab environments
* Practice detection, investigation, and remediation workflows
* Document lessons learned from security exercises

## Phase 5: Local AI Agent

* Build or deploy a local AI assistant
* Use the AI agent to help summarize lab notes
* Explore local AI for troubleshooting and documentation
* Experiment with security-focused workflows
* Keep sensitive data local where possible

## Long-Term Goals

* Build a repeatable and well-documented lab environment
* Improve cybersecurity, infrastructure, and automation skills
* Create portfolio-ready documentation
* Develop practical security projects that can be shared professionally
